# Ordered: They maintain the exact sequence in which you define them. The first item is always at index [0].
# Immutable: This is the big one. Once a tuple is born, you cannot add, remove, or change its elements. It’s "read-only."
# Heterogeneous: You can mix data types. A single tuple can hold a string, an integer, and a boolean all at once: ("Gemini", 3, True).
